‚≠ï What is Data Structure ?

‚Ü™ A data structure is a way of organizing and storing data in a computer so that it can be accessed and manipulated efficiently.

‚Ü™ It defines the relationship between the data, how the data is organized, and the operations that can be performed on the data.

‚Ü™ Data structures are essential for efficient algorithm design and are used in various applications, from simple programs to complex software systems.

Examples: arrays, string, linked lists, stacks, queues, trees, graphs, and hash tables.


‚≠ï Types of Data Structure

There are two types of Data Structure available.


1)  Linear Data Structure

‚Ü™ In a linear data structure, the elements are arranged in a sequential order, and each element is connected to its previous and next adjacent elements.

‚Ü™ Traversal in linear data structures is straightforward and follows a single path.

Examples: arrays, linked lists, stacks, and queues.


2)  Non-Linear Data Structure

In a non-linear data structure, the elements are not arranged in a sequential order.

‚Ü™ Instead, each element can be connected to one or more adjacent elements, forming a branching or hierarchical structure.

‚Ü™ Traversal in non-linear data structures can be more complex and may require recursive or iterative algorithms to visit all elements.

Examples: trees and graphs

‚Ü™ An algorithm is a step-by-step procedure or formula for solving a problem or accomplishing a task.

‚Ü™ Algorithms are used to perform calculations, data processing, and automated reasoning tasks.

‚≠ïAsymptotic Analysis of an algorithm

‚Ü™ Asymptotic analysis is a method used to analyze the efficiency of algorithms in terms of their time and space complexity as the input size increases.

‚Ü™ It provides a high-level understanding of how an algorithm scales with input size without getting into the details of specific hardware, software, or implementation.



Asymptotic Notations

‚Ü™ Asymptotic notations are used to describe the time complexity or space complexity of an algorithm as the input size grows.

‚Ü™ To compare efficiencies of algorithms, we use asymptotic notation, a mathematical tool that estimates time based on input size without running the code.

‚Ü™ It focuses on the number of basic operations in the program.


Types of Asymptotic Notations

There are three main asymptotic notations used in algorithm analysis.


1. Omega Notation (Œ©)

2. Big O Notation (O)

This notation represents the upper bound of the growth rate of a function.

For example, an algorithm with a time complexity of O(n) means that the time taken grows linearly with the size of the input, but not faster.
